The first Rule of ARIA

The W3C actually provides 5 general "rules" for using ARIA.

The first rule is probably the most important:

The First Rule of ARIA is...

...don't use ARIA!

(if you don't have to!)

What this rule is really about is getting you to use the native HTML element(s) with the correct semantics, and only fall back on ARIA if these don't exist. Native HTML gives you a lot "for free" that ARIA on its own doesn't, and is a lot harder to get wrong.

But there are still times where you may have to use it:

  • Many roles do not have native HTML equivalents - ARIA is the only option to give these more complex widgets the correct semantics for screenreaders.
  • You may have to support older browsers that don't have newer elements such as 
    header
    available.
  • Some native HTML elements have traditionally been very hard to apply styling to (eg select/checkbox). To produce the desired visuals it may be necessary to build a substitute with 
    div
    and/or 
    span
    elements and indicate the semantics using ARIA roles/attributes while adding the expected behaviour with Javascript.
