The second rule of ARIA

The second rule is: don't override native semantics.

This means: if you need to use an ARIA 
role
, put it on a 
div
or 
span
- elements which have no native role - rather than override the pre-existing role of some other element.
Example: <div role="alert"></div> is fine, but <h2 role="alert"></div> is questionable.

There is one exception:

It's OK to override semantics if the role you are adding is a more specialised meaning of the element's existing role. So for example <li role="menuitem"> (inside <ul role="menu">) is fine, as a menu role is a specialised type of list role.
