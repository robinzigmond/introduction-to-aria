The accessibility tree

See this blog post by Ben Myers for a great in-depth explanation of this topic.

Rather than an assistive technologies directly "reading" the HTML markup, and interpreting them in their own way, the markup is first converted by the browser into an "accessibility tree".

Why is this a good idea?

  • it provides a "common language" for all assistive technologies to use
  • assistive technologies are needed by their users to interpret everything on a computer/phone/tablet/etc, not just webpages! Operating systems provide Accessibility Trees for native features which assistive technologies already need to use - by reducing a webpage to an Accessibility Tree they need do nothing special for web pages

Note the different responsibilities here:

  • Converting the HTML/DOM into an Accessibility Tree is done by the browser
  • Interpreting the Accessibility Tree into user-facing behaviour is done by the assistive technology.

This explains why using different screenreaders on the same browser, and the same screenreader on different browsers, sometimes behave differently in the same situations.

Modern browser versions let you inspect the accessibility tree directly!
